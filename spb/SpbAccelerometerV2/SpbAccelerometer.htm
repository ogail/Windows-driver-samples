SpbAccelerometer

Summary

The SpbAccelerometer sample shows how to write a UMDF driver to control a peripheral device that is connected to a simple peripheral bus (SPB). In this sample, the peripheral device is a sensor (an accelerometer) and the SPB is an I2C bus. The SpbAccelerometer sample driver uses Windows features that are available starting with Windows 8 Consumer Preview.

The SpbAccelerometer sample driver is written for a sensor device that is permanently connected to the I2C bus. This sensor is not a Plug and Play device. Instead, the ACPI firmware for the hardware platform describes the sensor device's bus connection. The Plug and Play manager obtains the bus connection information from the ACPI driver, creates a connection ID to represent the bus connection, and passes the connection ID to the sample driver as a hardware resource. The sample driver uses the connection ID to open a logical connection to the sensor device, and obtains a handle to the connection. The driver specifies this handle as the target for I/O requests that it sends to the device.

To communicate with the sensor class extension, the SpbAccelerometer sample driver calls the methods in the ISensorClassExtension interface, and implements the ISensorDriver interface. These interfaces enable applications to use the Sensor API and Location API to communicate with the sample driver.Additionally, the sample driver implements an interrupt service routine (ISR) using passive-level interrupts that connects to the GPIO interrupt from the sensor device.

The SpbAcclerometer sample driver can send IRP_MJ_READ and IRP_MJ_WRITE requests to the sensor device. In addition, the driver can send IOCTL_SPB_XXX requests, which are defined in the Spb.h header file and supported by the SPB framework extension (SpbCx). SpbCx is available starting with Windows 8.

Installing the Sample

1.       Ensure that the driver builds without errors

2.       Copy the DLL and INF files to a separate folder

3.       Run devcon.exe update SpbAccelerometer.inf ACPI\<HWID> to install the driver on an existing device node. This node must declare I2C and GPIO resources in firmware. You can find this program in the tools\devcon folder where you installed the WDK.
